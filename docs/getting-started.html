Getting Started

Welcome to Oh My Codex. This guide will help you install the orchestration layer and run your first multi-agent workflow.

Prerequisite: You must have OpenAI Codex CLI installed and configured with your API keys.

#Installation

Oh My Codex (OMX) is distributed via NPM. You can install it globally to access the omx command anywhere.

1

Global Install

Install the core package using your preferred package manager.

bash 
npm install -g oh-my-codex
2

Run Setup

Initialize the environment, install required skills, and scaffold agent prompts.

bash 
omx setup

This will create a .omx/ directory in your project (or user home) containing state, logs, and memory.

3

Verify Health

Ensure everything is configured correctly, including tmux and model routing.

bash 
omx doctor

#Quick Start

Once installed, launch the OMX-enhanced Codex CLI. This will automatically attach a HUD and enable advanced orchestration.

bash 
omx

Core Concepts

Magic Keywords

Words like autopilot, ralph, and ulw trigger specialized execution modes automatically.

Skill Commands

Commands starting with $ (e.g., $plan, $team) invoke complex orchestrations.

Agent Prompts

Invoke specialized roles using /prompts:name (e.g., /prompts:architect).

#First Workflow Walkthrough

Let's try a typical "Full-Auto" workflow. This uses multiple agents to plan, execute, and verify a change.

1. Interactive Planning

Describe a complex task and let the planner agent interview you.

$plan "implement a secure JWT auth flow"

2. Review the Strategy

The Architect and Critic will refine the plan into actionable steps.

3. Parallel Execution

Once approved, OMX spawns workers in tmux to build the features in parallel. Supports mixed-provider teams (Codex, Claude, and Gemini) via OMX_TEAM_WORKER_CLI and OMX_TEAM_WORKER_CLI_MAP.

$team 3:executor "follow the approved plan"

4. Continuous Verification

The Verifier agent runs tests after every change to ensure zero regressions.
